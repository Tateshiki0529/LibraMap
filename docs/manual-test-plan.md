# LibraMap 実機テスト手順書

## 目的

LibraMap MVPが実際の返却業務に使える最低限の動作を満たしているか確認する。

対象範囲は、ISBN/JANスキャン、館内蔵書DB照合、NDL Search補助取得、NDC配架判定、GUI表示、58mmレシート出力である。

## 前提条件

- Windows PCで実施する
- Python仮想環境を作成済み
- `pip install -r requirements.txt` を実行済み
- バーコードスキャナがキーボード入力として動作する
- 実レシートプリンタ、またはシミュレーション出力で確認できる
- ネットワーク接続がある

## 使用機材

| 項目 | 内容 |
|---|---|
| PC | Windows |
| バーコードスキャナ | ISBN/JAN読み取り可能なもの |
| レシートプリンタ | ESC/POS対応 58mm |
| テスト資料 | ISBN-13付きの一般書、192系JAN付き資料、非対応JAN資料 |

## テスト前準備

1. 最新コードを取得する。

```powershell
git checkout master
git pull origin master
```

2. 依存ライブラリを確認する。

```powershell
.\.venv\Scripts\activate
pip install -r requirements.txt
```

3. 自動テストを実行する。

```powershell
python -m unittest discover -s tests
```

4. アプリを起動する。

```powershell
python -m libramap.main
```

## 基本動作テスト

### T-01 アプリ起動

1. `python -m libramap.main` を実行する。
2. メイン画面が表示されることを確認する。
3. 入力欄にフォーカスがあることを確認する。

期待結果:

- 画面タイトルが `LibraMap 返却支援` である
- 状態表示が `待機中` である
- プリンタ状態が表示されている

### T-02 ISBN-13 通常資料

1. `9784101010113` をスキャン、または入力してEnterを押す。

期待結果:

- `返却先を判定しました` と表示される
- 書名、NDC、返却先が表示される
- 返却先が `2階 / 書架 B-12 / 3段 / 4-6列` になる
- 右側マップで対象書架が強調表示される
- シミュレーション時は `data/receipts/` にPNGが保存される

### T-03 ISBN-13 禁帯出資料

1. `9784820414131` をスキャン、または入力してEnterを押す。

期待結果:

- `禁帯出資料` と表示される
- 禁帯出エリアへの返却案内が表示される
- マップ上で禁帯出エリアが強調表示される
- レシート画像に `禁帯出` が大きく表示される

### T-04 館内蔵書DB未登録資料

1. 館内DBに未登録のISBN-13をスキャンする。

期待結果:

- `要手動確認` と表示される
- `対象外資料` と表示される
- NDL Searchで書誌取得できた場合、書名またはNDCが補助表示される
- 配架先は自動判定されない
- レシート画像に手動確認の案内が入る

## JANテスト

### T-05 192系JAN

1. 192で始まるJANをスキャンする。
2. 追加スキャン待ち画面になったら、ISBN-13をスキャンする。

期待結果:

- 1回目の入力後に `追加スキャン待ち` と表示される
- ISBN入力後に通常の配架判定へ進む

### T-06 非対応JAN

1. 192以外のJANをスキャンする。

期待結果:

- `要手動確認` と表示される
- 非対応JANである旨が表示される
- 配架判定、印刷処理へ進まない

## プリンタテスト

### T-07 シミュレーション出力

1. プリンタ選択を `シミュレーション` にする。
2. 通常資料をスキャンする。
3. `data/receipts/` を確認する。

期待結果:

- PNGファイルが生成される
- 画像幅が58mm相当の縦長レシートになっている
- 書架コード、NDC、マップ、対象セルが読める

### T-08 実プリンタ出力

1. プリンタを接続する。
2. アプリ上で `共有プリンタ` または `USBプリンタ` を選ぶ。
3. 通常資料をスキャンする。

期待結果:

- レシートが印刷される
- 書架コードが大きく読める
- 対象セルが黒塗りで判別できる
- 用紙カットが動作する

補足:

- 実プリンタで失敗する場合は、まず `シミュレーション` で画像内容を確認する。
- Windows共有プリンタ名は現状 `\\localhost\RECEIPT` を前提としている。

## 異常系テスト

### T-09 無効な入力

1. `abc123` を入力してEnterを押す。

期待結果:

- バーコードを判定できない旨が表示される
- 手動確認を促す表示になる

### T-10 NDL Search通信失敗

1. ネットワークを切断する。
2. 未登録ISBNをスキャンする。

期待結果:

- アプリが落ちない
- 手動確認表示になる
- キャッシュが存在する場合は補助情報が表示される

### T-11 書架未定義NDC

1. `9784947017377` をスキャン、または入力してEnterを押す。

補足:

- 書名: ホームレス中学生
- JAN: `1920076013003`
- NDCはテスト用に `999.9` として登録している。

期待結果:

- 書架未定義として `要手動確認` になる
- アプリが落ちない
- レシート画像にも手動確認が表示される

## 終了確認

1. アプリを閉じる。
2. 以下を確認する。

```powershell
git status --short
```

期待結果:

- コード差分が出ていない
- `data/*.db` や `data/receipts/` はGit管理対象外である

## 判定基準

- T-01からT-07まで通れば、シミュレーションでのMVP動作は合格
- T-08まで通れば、実プリンタ込みのMVP動作は合格
- 異常系でアプリが落ちた場合は不合格
- 表示が読みにくい、レシートが判読しづらい場合は要改善
