# Agent Mandatory Startup Rules

These rules are mandatory for any coding task in this repository.

1. Read `docs/dev-memory.md` before starting work.
2. Follow the "GitHub本文運用ルール（恒久）" section in that file.
3. Follow the "ブランチ / PR / マージ運用ルール（恒久）" section in that file.
4. For GitHub Issue/PR/comment text:
   - Prioritize human-readable output.
   - Do not use inline escaped newlines like `\n` in final body text.
   - Use UTF-8 body files (`--body-file`) for create/edit flows.
5. After posting/updating Issue/PR body, verify rendered body text and fix immediately if garbled.

If these rules conflict with ad-hoc habits, these rules win.
